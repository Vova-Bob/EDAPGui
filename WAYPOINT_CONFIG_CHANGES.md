# Зміни у функціональності збереження waypoint файлів

## Опис
Додано можливість запам'ятовувати останній обраний JSON-файл маршрутних точок та автоматично завантажувати його при старті програми.

## Створено/модифіковано файли

### Нові файли:
- `gui_config.json` - конфігураційний файл для збереження останнього шляху до waypoint файлу
- `test_waypoint_config.py` - тестовий скрипт для перевірки функціональності

### Модифіковані файли:
- `EDAPGui.py` - основний файл GUI з доданими методами

## Додані методи в клас APGui (EDAPGui.py)

### `_get_gui_config_path()`
- Повертає шлях до конфігураційного файлу `gui_config.json`

### `_save_last_waypoint_file(filepath)`
- Зберігає шлях до останнього завантаженого файлу waypoint у конфігурацію
- Підтримує UTF-8 кодування для кириличних шляхів
- Додає версію програми до конфігурації

### `_get_last_waypoint_file()`
- Завантажує шлях до останнього файлу waypoint з конфігурації
- Обробляє помилки та повертає None у разі проблем

### `_load_last_waypoint_file()`
- Автоматично завантажує останній файл waypoint при старті програми
- Перевіряє існування файлу перед завантаженням
- Очищує конфігурацію, якщо файл не знайдено
- Логує результати операцій

## Модифіковані методи

### `open_wp_file()`
- Додано збереження шляху до конфігурації при успішному завантаженні файлу
- Очищення конфігурації при невдалому завантаженні

### `_refresh_waypoint_label()`
- Додано український коментар з описом функціоналу
- Відображає назву завантаженого файлу без шляху

## Виклик в конструкторі
- Додано виклик `self._load_last_waypoint_file()` в кінці конструктора для автоматичного завантаження

## Локалізація
Використовуються існуючі ключі локалізації:
- `ui.waypoint.loaded`: "Завантажено: {filename}"
- `ui.waypoint.no_list_loaded`: "<список не завантажено>"

## Обробка помилок
- Програма не падае при відсутності конфігураційного файлу
- Автоматично очищує невалідні шляхи з конфігурації
- Детальне логування всіх операцій

## Принципи KISS та DRY
- Логіка винесена в окремі невеликі функції
- Уникнення дублювання коду
- Простий та надійний підхід без зовнішніх залежностей

## Тестування
Створено тестовий скрипт `test_waypoint_config.py` для перевірки:
- Збереження шляхів до файлів
- Завантаження шляхів
- Обробки значень None
- Правильності роботи з файлами
